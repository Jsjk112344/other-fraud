<objective>
Build the deterministic classification layer: rules engine for obvious fraud/scalping cases and validation layer for catching contradictions and calibrating confidence. Both produce full VerdictResult instances indistinguishable from LLM output.

Purpose: CLAS-01 requires instant classification of obvious cases without LLM calls. CLAS-04 requires a safety net that catches LLM mistakes. Together they form the foundation that Plan 02 (LLM classification) builds on.
Output: Four Python modules (classify package) and comprehensive unit tests.
</objective>

    1. Create `backend/classify/validator.py` with `validate_verdict(verdict: VerdictResult, evidence: dict) -> VerdictResult`:
    - Import VerdictResult, Signal, ClassificationCategory, SignalSeverity
    - Import RULES_CONFIG from classify.config for threshold values (no magic numbers)
    - Maintain an `overrides: list[str]` accumulator

    **Contradiction check 1 -- LEGITIMATE + extreme underpricing:**
    - Extract price and face_value from evidence (with safe .get() chains)
    - If category is LEGITIMATE and price_ratio < RULES_CONFIG["extreme_underpricing"]["threshold"]:
      - Override category to LIKELY_SCAM
      - Set confidence to max(verdict.confidence, 85.0)
      - Update Pricing Anomaly signal to severity=CRITICAL, segmentsFilled=5
      - Append "Overridden from LEGITIMATE to LIKELY_SCAM -- pricing anomaly contradicts classification"

    **Contradiction check 2 -- LIKELY_SCAM + verified long-standing seller:**
    - Extract seller data: account_age_days, reviews_count
    - If category is LIKELY_SCAM and account_age_days > 365 and reviews_count > 50:
      - Reduce confidence by 20 points (min 40.0)
      - Append "Confidence reduced -- verified long-standing seller contradicts LIKELY_SCAM classification"

    **Confidence calibration -- high confidence with mixed signals:**
    - Define `_has_mixed_signals(verdict)`: returns True if signals contain both CRITICAL and CLEAR severities
    - If confidence > 95.0 and _has_mixed_signals: set confidence to 82.0, append "Confidence calibrated from {old}% to 82% -- mixed signals"
    - If confidence < 40.0 and all signals agree (all same severity direction): set confidence to 65.0, append "Confidence calibrated from {old}% to 65% -- consistent signals"

    **Apply overrides:**
    - If overrides list is non-empty: append " | Validation: " + "; ".join(overrides) to reasoning
    - Use verdict.model_copy(update={...}) for all mutations (Pydantic immutability)

    Helper `_update_signal(signals: list[Signal], name: str, severity: SignalSeverity, segments: int) -> list[Signal]`:
